\# Sample Candidates



This folder contains sanitized example vacation candidate records.



These files are safe to commit to GitHub because they are demonstration records, not raw connector payloads or private travel-booking data.



\## Purpose



The sample candidate shows the intended structure of a `vacation\_candidate` record.



A valid vacation candidate should combine:



\- One origin

\- One destination

\- One date range

\- One traveler profile

\- One flight option

\- One lodging option

\- One attractions/tours estimate

\- One safety/risk assessment

\- One total draft trip cost

\- One benchmark placeholder or benchmark estimate

\- Data quality flags

\- Candidate decision fields



\## Current Sample



File:



`sample\_lisbon\_candidate.yaml`



This sample represents a draft Lisbon vacation candidate for:



\- BOS to LIS

\- 2026-10-05 to 2026-10-16

\- 2 adults



\## Important Caveat



This sample is not a booking recommendation.



It should not be interpreted as live travel pricing, final scoring, or proof that the trip is undervalued.



The sample intentionally preserves caveats such as:



\- Flight price basis needs verification

\- Tripadvisor validation unavailable

\- Viator validation unavailable

\- Activity budget is placeholder-based

\- Fair-value benchmarking is not yet built
